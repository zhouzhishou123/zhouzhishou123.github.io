望而生畏的正则表达式

发表于 2020-05-30 更新于 2020-05-31

常用的元字符

  • 数量词（匹配x出现的次数）
    1. x* 0到多次
    2. x+ 1到多次
    3. x? 0或者一次
    4. x{n} n次
    5. x{n,} n次或者n次以上
    6. x{n,m} 至少n次至多m次
    7. x|y 匹配 x 或 y
  • 特殊元字符
    1. \ 转义字符
    2. . 匹配任意单个字符，但是行结束符除外：\n \r \u2028 或 \u2029
    3. \d 匹配任意阿拉伯数字。等价于[0-9]
    4. \D 匹配任意一个不是阿拉伯数字的字符。等价于[^0-9]
    5. \w 匹配任意字母、数字字符、下划线的字符。等价于 [A-Za-z0-9_]
    6. \W 匹配任意不是字母、数字、下划线的字符。等价于 [^A-Za-z0-9_]
    7. \s 匹配一个空白符，包括空格、制表符、换页符、换行符和其他 Unicode 空格
    8. \S 匹配一个非空白符
    9. \n 匹配一个换行
  • 字符集合
    1. [xyz] 字符组。匹配集合中的任意一个字符。可以使用连字符’-‘指定一个范围[a-z]
    2. [^xyz] 反义字符组,匹配任意不在括号内的字符,可以使用连字符’-‘指定一个范围[^a-z]
  • 边界字符
    1. ^x 匹配以x开始
    2. x$ 匹配以x结尾
    3. \b 匹配一个零宽单词边界,如一个字母与一个空格之间
    4. \B 匹配一个零宽非单词边界，如两个字母之间或两个空格之间
  • 分组
    1. (x) 匹配 x 并且捕获匹配项
    2. (?:x) 匹配 x 不会捕获匹配项
  • 断言
    1. x(?=y) 仅匹配被y跟随的x
    2. x(?!y 仅匹配不被y跟随的x

修饰符

  • i ignoreCase 忽略大小写匹配
  • m multiline 多行匹配
  • g global 全局匹配

正则的捕获

  • exec
  • test
  • match
  • replace

正则的分组和引用

正则捕获的贪婪性

  • 默认情况下，正则捕获的时候，匹配字符尽可能多的匹配，这种行为被称之为”贪婪”匹配。

  • 当我们使用正则表达式进行非贪婪匹配只需在带匹配的字符后跟随一个❓即可。

    let str = "今年是2020年",
    reg = /\d+?/g 
str.match(reg) // ['2', '0', '2', '0']

？常见的几种用法

  • x? 匹配0或者一次
  • ?? +? *? {1,5}? 取消捕获时的贪婪性
  • (?:) 只匹配不捕获
  • (?=) 正向预查
  • (?!) 反向预查
  • (?<=y)x x只有在y后面才匹配
  • (?<!y)x x只有不在y后面才匹配
